=== HOW IT WORKS ===

1. AUTHENTICATION FLOW:
   User → Sign-up → Better Auth creates session → Redirects to app

2. PROTECTED ROUTES:
   Route handler → Check session → Allow/redirect to /sign-in

3. SERVER ACTIONS:
   Client → Server Action → getUserId() → Query with userId scope → Response

4. DATABASE QUERIES:
   Every query includes: where(eq(table.userId, userId))
   This prevents users from seeing each other's data

=== TROUBLESHOOTING ===

Server not starting?
  → Check: npm list better-auth pg drizzle-orm
  → Restart: npm run dev

Database connection error?
  → Verify DATABASE_URL in env vars
  → Check Neon dashboard for active connections

Sessions not working?
  → Ensure BETTER_AUTH_SECRET is set (≥32 chars)
  → Clear browser cookies and sign up again

Questions?
  → Read: SETUP_GUIDE.md
