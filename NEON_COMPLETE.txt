DATABASE SCHEMA
================================================================================

TABLE: comments
  id SERIAL PRIMARY KEY
  comment TEXT NOT NULL
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

TABLE: transactions
  id SERIAL PRIMARY KEY
  user_id VARCHAR(255) NOT NULL
  amount DECIMAL(10, 2) NOT NULL
  description TEXT
  type VARCHAR(50)
  status VARCHAR(50) DEFAULT 'pending'
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

TABLE: users
  id SERIAL PRIMARY KEY
  email VARCHAR(255) UNIQUE NOT NULL
  name VARCHAR(255)
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

(All tables auto-created on first use with IF NOT EXISTS checks)

TROUBLESHOOTING
================================================================================

ISSUE: DATABASE_URL is not set
SOLUTION: Add DATABASE_URL to .env.local and restart dev server

ISSUE: Connection refused
SOLUTION: Verify Neon credentials and project status

ISSUE: Table does not exist
SOLUTION: Call POST /api/db-init to create tables

ISSUE: Form not submitting
SOLUTION: Check browser console for errors (all logged with [v0])

ISSUE: No data showing
SOLUTION: Ensure tables were created and data was inserted

For more help: See NEON_SETUP.md → Troubleshooting section
